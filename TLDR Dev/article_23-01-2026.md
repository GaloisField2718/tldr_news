 SCALING POSTGRESQL TO POWER 800 MILLION CHATGPT USERS (14 MINUTE
READ) [5] 

 OpenAI scaled PostgreSQL to support 800 million ChatGPT users and
millions of queries per second for its predominantly read-heavy
workloads. To achieve this, its team offloaded write-heavy operations
to sharded systems like Azure Cosmos DB and implemented application
and query optimizations. Their architecture uses a single primary
Azure PostgreSQL instance and nearly 50 global read replicas. 

 WHAT IS A DESIGN ENGINEER? (5 MINUTE READ) [6] 

 A design engineer bridges the gap between design and frontend
development, creating software that not only works perfectly but also
feels great to use. They directly implement designs in code, build and
maintain design systems, and add in the critical final details of UI
polish, like micro-interactions and animations. 

 WHY DOES SSH SEND 100 PACKETS PER KEYSTROKE? (9 MINUTE READ) [7] 

 A developer building a high-performance SSH game found that a single
keystroke generated over 100 small packets. The root cause was
identified as OpenSSH's keystroke timing obfuscation feature, which
sends "chaff" packets to disguise typing patterns. By patching Go's
SSH library to disable the `ping@openssh.com` extension server-side,
the developer achieved a dramatic >50% reduction in CPU and bandwidth.
LLMs assisted in the debugging process.

 KEEPING 20,000 GPUS HEALTHY (11 MINUTE READ) [12] 

 Modal runs a globally distributed pool of over 20,000 GPUs across
multiple cloud providers. Its approach begins with instance type
testing and selection, evaluating performance and reliability
differences between cloud hyperscalers. This is followed by machine
image preparation and automated boot checks to guarantee GPU health
before customer workloads begin. Throughout an instance's life, Modal
performs continuous passive and active healthchecking. 

 THE CONTEXT COLLAPSE PROBLEM (11 MINUTE READ) [13]

underperform on legacy codebases, a "context collapse problem" caused
by the invisibility of institutional knowledge and historical
decisions. Good solutions include treating documentation as
infrastructure, incrementally restructuring code for AI legibility,
