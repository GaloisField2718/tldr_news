# Production illustration v2

Rendering style and story-specific editorial concept are separate. Blind cross-story review selected `print-graphic-v1`: it produced the preferred Secure Sandboxes and Hyundai images, while constructed collage was more distinctive but less reliable and produced a visible-text hard rejection. AMD concepts remain calibration-only and are never universal production concepts.

`production-v2` locks a rendering direction only: authored print language, simplified shapes, restrained palette, physical texture, hierarchy, negative space, and controlled abstraction. It excludes CGI, cyberpunk, monumental racks, pseudo-technical detail, and all text-like artifacts, without excluding legitimate people, objects, depth, or environments needed by a story.

Production visual briefs are versioned. V2 requires `schema_version: "2.0.0"`, `mode`, exact `sources`, `editorial_idea`, `central_subject`, `visual_relationship`, `composition`, ordered unique `literal_elements`, `abstraction_level` (`low`, `medium`, or `high`), ordered unique `forbidden_elements`, ordered unique `failure_modes`, and factual `alt_text`. Text fields are bounded (300–500 characters by field); literal and failure arrays cap at 12 and forbidden elements at 20. Historical v1 briefs remain readable under their old contract but cannot be silently upgraded.

V2 artifacts explicitly store `illustration_profile_id: "production-v2"` and `visual_brief_schema_version: "2.0.0"`; v1 artifacts omit both. Mixed contracts are invalid. The canonical v2 illustration hash includes immutable profile identity/content, prompt and brief versions, every ordered brief field, exact source references/titles/summaries, image model, and normalization configuration. It excludes timestamps, request IDs, usage/cost, status/failure bookkeeping, URLs, object keys, stored image metadata, Git metadata, and manifest order.

Historical validity and active-contract currency are separate decisions. A v1 artifact can validate under its stored prompt/hash contract while remaining stale against active prompt `2.0.0`, profile `production-v2`, and brief schema `2.0.0`. Preflight reports validity, compatibility, staleness reasons, refresh/regeneration requirements, bounded expected calls, and no-op status independently.

The safe live migration path is **Option A**: the valid but incompatible 2026-07-21 v1 brief reports `prompt_version_mismatch`, `illustration_profile_mismatch`, `visual_brief_schema_mismatch`, and `incompatible_visual_brief`, then requires one editorial call for a strengthened AMD v2 brief, followed by at most one image call and one immutable WebP upload. The old R2 object is preserved. A second identical run is a byte-preserving zero-call no-op. A compatible v2 failed image remains brief-compatible and can be reused only under existing explicit retry semantics; persisted failures do not create an automatic retry loop. URL-only changes and non-semantic metadata do not make the semantic contract stale. Force remains explicit and never bypasses v2 validation.

Provider accounting distinguishes request attempts from successful responses for editorial and image stages. Sanitized HTTP diagnostics retain status, bounded provider code/message, safe request ID, and retryability without headers, credentials, prompts, or environment values. The strict structured-output contract uses the provider-supported single-value `enum` for schema version; unsupported schema keywords are rejected locally before a request.

A migration from an existing valid published artifact is transactional at the official repository boundary. Editorial refresh, brief validation, image creation, normalization, immutable upload, and verification complete before the atomic artifact/manifest replacement. Any editorial, image, validation, or storage failure preserves both official JSON files byte-for-byte and reports failure out-of-band. Deterministic fallback may still be written for a genuinely missing edition, but never replaces a valid published artifact during migration. After an HTTP 400, inspect sanitized diagnostics, fix and test the request contract, and require a new explicit authorization before another live attempt.

Prompt restrictions reduce but cannot guarantee text/artifact absence. Optional second-stage OCR or visual moderation remains future work and is not invoked by this pipeline.

Rollback is a reviewed change restoring `baseline-v1` and its previous prompt version; do not overwrite historic artifacts or R2 objects.
