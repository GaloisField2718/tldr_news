ATTACKS & VULNERABILITIES

 CL-STA-0969 INSTALLS COVERT MALWARE IN TELECOM NETWORKS DURING
10-MONTH ESPIONAGE CAMPAIGN (5 MINUTE READ) [5] 

 Chinese state-sponsored threat actor CL-STA-0969 conducted a 10-month
espionage campaign that targeted Southeast Asian telecom networks. The
campaign used custom malware, including AuthDoor, GTPDOOR, and
ChronosRAT, to enable remote control and location tracking. The group
deployed advanced evasion techniques, including DNS tunneling and
traffic routing through compromised mobile operators, while
maintaining high operational security. There was no evidence of data
exfiltration despite the attackers installing multiple backdoors and
scanning tools across critical telecommunications infrastructure. 

 CRITICAL ALERT: AKIRA RANSOMWARE SURGE TARGETS SONICWALL FIREWALLS (3
MINUTE READ) [6] 

 Akira ransomware attacks targeting SonicWall SSL VPN connections have
surged since July. Exploiting CVE-2024-40766 (CVSS 9.3) and unpatched
SonicOS devices with disabled MFA, attackers rapidly escalate from VPN
access to system encryption within hours, affecting hundreds of
thousands of devices across multiple SonicWall generations.
Organizations must immediately patch to the latest firmware, enable
MFA on all VPN accounts, and restrict firewall management access to
trusted IP ranges. 

 ATTACKERS EXPLOIT LINK-WRAPPING SERVICES TO STEAL MICROSOFT 365
LOGINS (2 MINUTE READ) [7] 

 Cloudflare's Email Security team observed a new campaign in which
threat actors abuse link-wrapping services from Proofpoint and
Intermedia to phish users for Microsoft 365 credentials. Attackers
abused Proofpoint link-wrapping in various ways, including
multi-tiered redirect abuse with URL shorteners via compromised
accounts. In the Intermedia campaign, threat actors sent a message
impersonating a ‚ÄúZix‚Äù secure message notification or a Teams
notification that would redirect users to a fake login page to harvest
Teams credentials.

STRATEGIES & TACTICS

 EXPLOITING TRUST: HOW SIGNED KERNEL DRIVERS FUEL MODERN KERNEL LEVEL
ATTACKS ON WINDOWS (11 MINUTE READ) [8] 

 An emerging pattern in kernel level attacks is to use a signed kernel
loader to load a signed second stage kernel driver or reflectively
load an unsigned kernel driver. These signed drivers are often signed
via the Windows Hardware Compatibility Program (WHCP). The WHCP
requires an extended validation (EV) certificate, which is designed to
increase the security of the program. However, the EV certificate
application process requires little human interaction, and there are
dark web sellers of EV code-signing certificates. 

 KUBERNETES SECURITY FUNDAMENTALS: AUTHORIZATION (7 MINUTE READ) [9] 

 Kubernetes handles authorization by utilizing multiple authorization
providers in the cluster requested in the order specified to the
kube-server as flags on startup. Each provider can explicitly approve
or deny the request or respond with ‚Äúno opinion.‚Äù If all providers
respond ‚Äúno opinion,‚Äù the request is denied. Providers include
AlwaysAllow and AlwaysDeny, which allow or deny every request; Node
Authorizer, which restricts the access of the kubelet user;
Attribute-Based Access Controls (ABAC), which enables cluster-admins
to define authorization rules based on the request's characteristics
statically; Role-Based Access Controls (RBAC), which is the most
commonly used provider and maps permissions to roles; and Webhook,
which is widely used in cloud provider clusters like AKS, GKE, and
EKS.

 PLAYBOOK-NG (GITHUB REPO) [13] 

 Playbook-NG is a stateless web-based application that matches
incident findings with countermeasures for adversary containment and
eviction. Users can export the results in numerous formats such as
JSON, Microsoft Word, Excel, and Markdown. The application does not
save information about the users or their inputs. Work is cleared when
the user browses off or clears the playbook.
