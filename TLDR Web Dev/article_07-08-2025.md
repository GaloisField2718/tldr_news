ARTICLES & TUTORIALS

 HOW WE BUILT THE WORLD'S FASTEST VIN DECODER (4 MINUTE READ) [5] 

 Cardog built the world's fastest VIN decoder by optimizing the
NHTSA's VPIC database, which contains 40 years of vehicle data but
doesn't have performance optimization. It achieved this by stripping
regulatory metadata, removing unused patterns, and applying modern
SQLite optimization techniques, reducing the database size from 1.5GB
to 21MB. This resulted in a huge performance improvement, with
response times decreasing from 3.2 seconds to under 30ms and
eliminating network dependency. 

 PARSING PROTOBUF LIKE NEVER BEFORE (45 MINUTE READ) [6] 

 hyperpb is a new high-performance and runtime-dynamic Protobuf
parsing library for Go. It uses a dynamic approach, compiling code at
runtime to enable online profile-guided optimization for parsing
efficiency. This article goes over its architecture, focusing on its
VM-based parser, memory management, and unique optimizations like
zerocopy strings and arena reuse.

 DEVELOPERS, NOT OPERATORS (7 MINUTE READ) [7] 

 AI is not reinventing developers, and adopting AI tools is not the
only path to success. It's important that devs have the freedom to
choose their own tools and approaches. Software development is more
than just generating code quickly - it requires understanding
problems, designing solutions, and making architectural decisions. 

 HTML IS DEAD, LONG LIVE HTML (16 MINUTE READ) [8] 

 The DOM and CSS are outdated and inefficient for modern web apps due
to their complexity, legacy issues, and the blurring of UI and
content. Common problems include the stringly-typed nature of the DOM,
CSS' inside-out approach to layout, and the inadequacy of text
manipulation APIs. Alternatives like "HTML in Canvas" and GPU-based
renderers are promising to look into. 

 IT'S TIME FOR MODERN CSS TO KILL THE SPA (8 MINUTE READ) [9] 

 Modern CSS features like the View Transitions API and Speculation
Rules have made Single Page Applications (SPAs) largely unnecessary by
enabling smooth page transitions and instant navigation without
requiring heavy JavaScript frameworks. While SPAs were originally
adopted to create app-like experiences, browsers can now do what SPAs
do natively with simple CSS.

 WRITING A RUST GPU KERNEL DRIVER: A BRIEF INTRODUCTION ON HOW GPU
DRIVERS WORK (8 MINUTE READ) [13] 

 This article teaches the basics of GPU driver architecture, focusing
on the user-mode driver (UMD) and kernel-mode driver (KMD)
relationship using an example of rendering a rotating cube with Vulkan
(VkCube). The UMD translates high-level API commands into lower-level
GPU instructions, while the KMD manages memory allocation, job
scheduling, and hardware initialization. The Tyr driver, a Rust-based
KMD, provides an interface for device information, memory management,
job submission, and tiler heap management. 

 WHY BUILDING BILLING SYSTEMS IS SO PAINFUL (10 MINUTE READ) [14] 

 There are many hidden complexities of building billing systems, far
beyond simple subscription models. Billing involves managing various
pricing models, real-time usage tracking, and adapting to changing
needs. Billing is not just invoicing, but a critical infrastructure
that impacts the entire business, so it often requires a specialized
team with expertise in both engineering and finance.
