ARTICLES & TUTORIALS

 OPENPOKE: RECREATING POKE'S ARCHITECTURE (14 MINUTE READ) [4] 

 This dev reverse-engineered Poke, a popular iMessage assistant
chatbot, and developed OpenPoke, a working prototype of its
multi-agent architecture. OpenPoke uses an Interaction Agent to manage
specialized Execution Agents that handle tasks like email management
and setting reminders. The system incorporates multi-tiered memory,
including conversation logs, agent logs, and email as external truth.
The assistant separates personality from execution, while recognizing
personality as an important product element. 

 NPM SECURITY BEST PRACTICES (13 MINUTE READ) [5] 

 This is a consolidated list of best practices for securing against
incidents in the NPM ecosystem, covering npm, bun, deno, pnpm, and
yarn. For devs, important best practices include pinning dependency
versions, including lockfiles in repos, disabling lifecycle scripts,
and reducing external dependencies. Maintainers should enable 2FA,
create tokens with limited access, generate provenance statements, and
always review published files. 

 FINE-GRAINED HTTP FILTERING FOR CLAUDE CODE (5 MINUTE READ) [6] 

 httpjail is a tool designed to mitigate security risks associated
with increasingly powerful coding agents by implementing fine-grained
HTTP(S) filtering. It works by intercepting HTTP requests and applying
user-defined JavaScript or shell script rules to control agent access
to external resources, focusing on preventing destructive actions,
data leaks, and excessive authority. It has both a "weak" mode that
relies on environment variables and a "strong" mode that uses network
namespaces, including TLS interception for HTTPS traffic inspection.

 DEAR GITHUB: NO YAML ANCHORS, PLEASE (9 MINUTE READ) [7] 

 This dev isn't a fan of GitHub's recent addition of YAML anchor
support to GitHub Actions. YAML anchors are redundant, complicate
workflow comprehension for both humans and machines, and don't have
any unique utility due to the absence of merge key support in GitHub
Actions. This change makes it harder to analyze workflows for
vulnerabilities, thereby worsening GitHub Actions' security. 

 WHAT HAPPENS WHEN CODING AGENTS STOP FEELING LIKE DIALUP? (8 MINUTE
READ) [8] 

 Current coding agents' speed and performance is similar to the slow
and unreliable dial-up era of the internet. There are issues with
reliability and a strain on infrastructure due to exploding AI token
usage. However, with time, there will be faster token processing
speeds, which will enable more unsupervised and efficient coding agent
workflows.

 CAP'N WEB: A NEW RPC SYSTEM FOR BROWSERS AND WEB SERVERS (21 MINUTE
READ) [11] 

 Cap'n Web is a new, open-source RPC system for JavaScript
environments for facilitating communication between applications over
a network. Unlike traditional RPC, Cap'n Web supports bidirectional
calling, object-capability security, and promise pipelining, aiming
for a developer experience similar to native JavaScript APIs. It makes
setup easier, integrates with TypeScript, and uses human-readable JSON
serialization.

 COMPILEBENCH: CAN AI COMPILE 22-YEAR-OLD CODE? (8 MINUTE READ) [13] 

 CompileBench is a new benchmark created to test the ability of LLMs
to handle complex software development tasks like dependency
management and legacy code compilation. The benchmark tested 19 LLMs
on 15 real-world tasks, including cross-compiling and resurrecting old
code, showing varying levels of success and identifying issues like
cheating. Anthropic models performed best in overall success and
speed, while OpenAI models were the most cost-efficient. 

 WHY LOCAL-FIRST APPS HAVEN'T BECOME POPULAR? (6 MINUTE READ) [14] 

 Local-first apps have adoption challenges due to the complexity of
syncing data across devices, especially offline. The key to successful
local-first apps lies in achieving eventual consistency, making sure
all devices converge to the same state without data loss. This happens
through techniques like Hybrid Logical Clocks (HLCs) for reliable
ordering and Conflict-Free Replicated Data Types (CRDTs) to handle
data conflicts.
