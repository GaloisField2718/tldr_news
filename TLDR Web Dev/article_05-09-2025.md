ARTICLES & TUTORIALS

 WHAT IF OPENDOCUMENT USED SQLITE? (20 MINUTE READ) [4] 

 SQLite can be used as the foundation for file formats like
OpenDocument Presentation (ODP) instead of the current ZIP archive
structure. SQLite has advantages such as smaller file sizes, faster
file saving and startup times, and reduced memory usage due to its
ability to store and retrieve data in smaller, incremental pieces.
Furthermore, SQLite allows for features like document versioning. 

 MEMORY IS SLOW, DISK IS FAST (24 MINUTE READ) [5] 

 Memory is not always faster than disk, as disk I/O can outperform
memory caching in specific scenarios. Experiments involving counting
integers in a large dataset show that traditional memory access
methods like `mmap()` introduce overhead due to memory latency. By
using an `io_uring` based I/O engine to stream data directly from
disk, faster data processing is achieved compared to accessing data
from the page cache. 

 HOW WE BUILT AN INTERPRETER FOR SWIFT (A COMPILED LANGUAGE) (5 MINUTE
READ) [6] 

 Bitrig built a Swift interpreter to dynamically run Swift apps on
iPhones without requiring compilation and signing through Xcode. The
interpreter uses SwiftSyntax to parse Swift code and represents
runtime values using an `InterpreterValue` enum, handling both native
and custom types. For framework APIs like SwiftUI, it pre-compiles
calls to initializers and other functions, parsing `.swiftinterface`
files to automate the process.

 SAYING NO IS NOT A FREE ACTION IN THE WORLD OF SOFTWARE ENGINEERING
(6 MINUTE READ) [9] 

 Saying no in software engineering, while important for work-life
balance, comes with psychological and social costs, making it harder
than it seems. People often avoid refusing requests due to negative
connotations, potential career risks, and the desire to please others.
Instead of simply telling people to "just say no," we should
understand the burden of refusal and create a supportive culture that
encourages saying no when necessary.

 OPEN SWE (GITHUB REPO) [11] 

 Open SWE is an open-source, cloud-based coding agent built with
LangGraph that autonomously understands codebases and implements
solutions, from planning to pull requests. It has human-in-the-loop
feedback, parallel task execution, and automatic issue/pull request
management. Users can initiate tasks via a UI or directly from GitHub
issues with specific labels.

 A PM'S GUIDE TO AI AGENT ARCHITECTURE: WHY CAPABILITY DOESN'T EQUAL
ADOPTION (12 MINUTE READ) [14]

integration, skills & capabilities, and evaluation & trust. Most AI
agent flows should start with a simple, single-agent architecture and
prioritize graceful escalation to humans over striving for constant
correctness to build user trust.
